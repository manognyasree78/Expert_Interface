In the *chat panel* of expert pages:

* Clicking the *Attach* button should allow:

  * Document file selection (.pdf, .docx, .txt)
  * Image upload (.png, .jpg)
* Show selected file name or preview in the chat bubble before/after message.

💡 You can use <input type="file" multiple /> styled with custom icon for file upload.

### ✅ *Fix #3: Multi-question Answering in Expert Page*

*Expected behavior*:

* Every time the user types a new question:

  * It is added to the chat panel
  * The *right preview panel* generates an answer and appends it *below the previous answer*
* Each question → 1 answer → appended sequentially (in same scrollable thread)

✅ Answers must *not overwrite* each other.
✅ Scrollable thread grows vertically.
✅ Ensure both panels scroll independently.

---

### ✅ *Fix #4: Persistent Chat + Answer Thread*

* When a user leaves the expert page (e.g., via Back to Main)
* Then re-enters the *same domain expert page*:

✅ Show full *chat thread history*
✅ Show all *answers* from right preview panel (in the same order as before)
✅ Use localStorage or sessionStorage to preserve thread and answers

### ✅ *Fix #5: Bell Icon Functionality (Inbox View)*

On expert page (top left next to Expert name):

* Clicking the 🔔 Bell icon opens a *popup modal*:

  * Show list of:

    * User Questions
    * Corresponding Expert Answers (in preview format)
* Style like a mini inbox → scrollable popup

### ✅ *Fix #6: Main Page Question Bar Behavior*

*Previous issue*: User had to retype question after redirection
*Now behavior*:

* User types question in /dashboard → system detects domain:

  * If Python → redirect to /expert/python
  * If E-commerce → redirect to /expert/ecommerce

* On redirection:
  ✅ Show question in chat panel
  ✅ Show matching answer in preview panel directly
  ✅ No need to retype question

💡 To do this: pass question as state via router or sessionStorage → then render both on expert page mount.
